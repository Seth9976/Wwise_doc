Platform-Specific Setup

This section contains setup instructions specific to certain platforms.

Android: Linking

For integration into your app, the libSpeechEngine module must be linked. How you do this depends on your configuration and build systems.

As an example, you can add this to your app's Android.mk file:

include $(CLEAR_VARS)
LOCAL_MODULE := libSpeechEngine
LOCAL_SRC_FILES := $(APP)/$(CONFIG)/bin/libSpeechEngine.so
include $(PREBUILT_SHARED_LIBRARY)

Add the libSpeechEngine module as a dynamic or shared library to the configurations as needed.

Android: Dependencies

Depending on your choice of voices, the speechEngine module must load a dynamic library for each language. For example, when using Adam, the vt_bre.so (British English) module is used.

These libraries follow the vt_[lang].so naming convention and need to be in the same folder as libSpeechEngine.so (usually the jniLibs folder).

When installing the plug-in, these voice files are copied into $WWISEROOT/SDK/Android_{arch}/{config}/{config}/libs.

iOS: Compiling the speechEngine IntegrationDemo page

To compile the speechEngine plug-in page of the IntegrationDemo, you need to modify the Xcode project included with the Wwise SDK.

In the IntegrationDemo Xcode properties:

  • Under General - Frameworks, Libraries and Embedded Content:

    • Add all the vt_[lang]_q22.framework files found under Applications/Audiokinetic/Wwise[version]/SDK/iOS_Xcode[version]/[configuration]/bin.

  • Under Build Settings:

    • Add @executable_path/Frameworks to Runpath Search Paths.

    • Add ../../../iOS_Xcode$(XCODE_VERSION_MAJOR)/Profile$(EFFECTIVE_PLATFORM_NAME)/bin to Framework Search Paths.
