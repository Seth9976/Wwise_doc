Containers hierarchy: music objects

Interactive music involves a hierarchical series of Music Segments and Music Tracks, which you can organize in various music containers. This is done in the Containers hierarchy.

Each music object has a set of properties and behaviors that can be used to define how the objects passing through the object will sound and play in game. These properties and behaviors are located in the Properties and Contents Editor.

The descriptions of each of the music object property options can be found in the following Property Editor descriptions:

  • Property Editor: Music Switch Container

  • Property Editor: Music Playlist Container

  • Property Editor: Music Segment

  • Property Editor: Music Track
