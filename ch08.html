Managing platforms

A game is often released for multiple platforms. You can specify the target platforms using the Platform Manager in Wwise. You can also use the Platform Manager to specify volume thresholds and max voice instances for each platform.

In the Wwise menu bar, select Project > Platform Manager.

A platform consists of a platform name and a base SDK. You can specify as many platforms as desired for any listed SDK.

Wwise includes a list of base platforms which cover the primary SDKs used with Wwise, and you are free to create your platforms according to all the base platforms pertinent to your project. Additionally, you might want to add multiple platforms with the same base platform to accommodate differences. For example, you may have both iPhone and iPad platforms to develop in your project. They're not exactly the same even though they're fundamentally iOS platforms. To facilitate your management of these distinct project types, you could create two iOS-based platforms: one specific to iPad and another specific to iPhone.

Platform Manager dialog

Use the Platform Manager dialog to define the project's platforms.

The Platform Manager queues all requested platform actions (Add/Remove/Rename). Click OK to execute all pending platform actions.

Note

After you click OK, Wwise saves and then reloads the project.

The following table describes the interface elements in the Platform Manager dialog.

Interface Element

Description

Right-click the table header to open the Configure Columns dialog where you can specify which columns to display and their order. See Using tables.

Platform

The unique alphanumeric name of the defined platform.

Base Platform

The platform's SDK.

Pending Actions

Actions, such as Add or Copy Settings for the specified platform that you complete by clicking OK.

Volume Threshold

The default volume level below which a voice will be managed by behaviors defined in the Advanced Settings tab of the Property Editor. For example, voices that fall below the volume threshold can do any of the following:

  • Continue to play

  • Be killed

  • Sent to the virtual voice list

This value can be overridden by using the Wwise SDK API.

Note: Even if it continues to play or becomes virtual, the voice will be inaudible while it’s still below the volume threshold. Note: The default slider range is from -96 to 0. You can go over those limits by entering the value directly, or by rolling the mouse while the focus is on the edit control.

For more information, see Specifying volume thresholds for a project .

Default value: -80

Range: -96 to 0

Max Voice Instances

The maximum number of simultaneous voices that can be active at the same time in the project. Virtual voices don't count as active voices. Over this limit, voices with the lowest priority adopt their virtual behavior. If priorities are equal, older sounds have a higher priority. Each sound can override its own virtual behavior, which can be:

  • Continue to play

  • Be killed

  • Send to virtual voice

Note

The Maximum number of voice instances might not be respected if you set the virtual behavior on some sounds to Continue to play. To have this limit respected, you must set the virtual behavior of playing voices to Kill voice or Send to Virtual voice.

You can override this value using the Wwise SDK API.

See Specifying max voice instances.

Adding platforms

The following table describes the interface elements in the Add Platform dialog.

Interface element

Description

Base Platform

The SDK for the platform, selected from a comprehensive list of SDKs that can you can use with Wwise.

Override Default Name

The unique alphanumeric name given to the new platform. Using the default name is strongly recommended because third-party Wwise integrations might reference that platform name.

Note

You can use underscores but not spaces when you rename a platform.

Copy Settings

If enabled, the new platform inherits settings from the selected platform. You can only copy settings from an existing platform. For more details, see Platform-specific settings that are copied over from the source to the target platform:.

Click to close the Add Platform dialog. The platform is added to the Platforms list of the Platform Manager with a pending Add action.

Click to close the Add Platform dialog without making any changes to your project.

To add a platform:

  1. In the Platform Manager, click Add.

    The Add Platform dialog appears.

    Note

    If a platform is highlighted in the Platform Manager when you click Add, the fields are populated with that platform's default information.

  2. Select a Base Platform from the list of SDKs.

  3. To specify a new name for the Platform, select Override Default Name and then enter the new name. It must be alphanumeric (no spaces, but underscores are accepted) and unique among all defined platform names.

    Tip

    Avoid overriding the default name as third-party Wwise plug-ins may reference it.

  4. Optionally, copy the settings from an existing platform by enabling the Copy settings from check box and then selecting the desired platform.

  5. Click OK to confirm the new platform information and close the Add Platform dialog.

    The new platform name, base platform, and pending Add action appear in their respective columns in the row containing the new platform.

    Note

    To remove this platform instantly, click Remove.

  6. Adjust the volume threshold or max voice instances for your platform if desired. See Specifying volume thresholds for a platform.

  7. Click OK to add the new platform to your project.

    A warning appears explaining that these platform changes can't be undone.

  8. Click Yes to proceed with the pending action.

    Another warning appears to remind you to verify the platform changes, especially in path names, after the project reloads.

  9. Click OK to reload the project.

    The project closes and reloads with the new platform settings. Depending on the size of your project, this may take a few seconds.

Copying platform settings

Use the Copy Platform Settings dialog to copy the settings from one platform to another. This is typically used when adding a new platform to your project. This could save you a lot of time if most of the settings from the source platform are the same as the destination platform you are adding.

The following table describes the interface elements in the Copy Platform Settings dialog.

Interface element

Description

From

The platform containing the settings you want to copy. You can only copy the settings from a platform that existed before you opened the Platform Manager. In other words, you can't copy the settings from a platform that has a pending Add action.

To

The platform to which you want to copy the settings. This might be either a newly added platform—meaning a platform that didn't exist before you opened the Platform Manager—or an existing platform that isn't the source of a settings copy.

Closes the Copy Platform Settings dialog and adds the Copy settings from <name of platform> action to the platform.

Closes the Copy Platform Settings dialog without making any changes to your project.

To copy platform settings:

Warning

This operation can't be undone, and it clears the Wwise-related contents on your clipboard and your Undo history. Be sure to have an up-to-date backup of your project in case you decide to revert these changes later.

  1. In the Platform Manager, click Copy settings.

    The Copy Platform Settings dialog appears.

  2. Select the source platform in the From list.

  3. Select the destination platform in the To list.

  4. Click OK to commit the newly added platforms.

    A warning appears explaining that these platform changes can't be undone.

  5. Click Yes to proceed with the pending actions.

    Another warning appears to remind you to verify the platform changes, especially in path names, after the project reloads.

  6. Click OK to reload the project.

    The project closes and reloads with the new Platform-specific settings that are copied over from the source to the target platform:. Depending on the size of your project, this may take a few seconds.

Removing platforms

  1. In the Platform Manager, select the platform you want to remove.

  2. Click Remove.

    The Remove action appears in the Pending Actions column for the platform.

    Note

    To cancel the Remove action, you can close the Platform Manager or click Cancel.

  3. Click OK to activate the removal of the selected platforms.

    A warning appears explaining that these platform changes can't be undone.

  4. Click Yes to proceed with the pending action.

    Another warning appears to remind you to verify the platform changes, especially in path names, after the project reloads.

  5. Click OK to reload the project.

    The project closes and reloads with the new platform settings. Depending on the size of your project, this may take a few seconds.

Specifying volume thresholds for a platform

The Volume Threshold refers to the point below which a voice will be managed by behaviors defined in the Advanced category of the Property Editor. You can specify a default volume threshold in your project in the Platform Manager dialog. This value will be used if the Volume Threshold is not specifically set using the Wwise API. For more information on managing low level sounds and motion objects and defining the object's behavior when it reaches the volume threshold, refer to Managing low-volume objects.

Use the following step list to define the volume thresholds for a platform.

To define the volume thresholds for a platform:

  1. Open the Platform Manager dialog by doing one of the following:

    • From the Project menu, select Platform Manager.

    • Press Shift+Alt+P.

  2. For each platform, specify the point at which the sound will be managed by the behaviors defined in the Advanced category of the Property Editor.

    Note

    This default value will be used if no Volume Threshold setting is specified using the Wwise API.

  3. Click OK to save your settings and to close the Platform Manager dialog.

Specifying max voice instances

Much like specifying the platform's volume thresholds, within a project there is a specified limit of voice instances that can play at one time. If the project exceeds this specified value of voice instances, then it reverts to the Advanced Settings of the playing objects as defined in the Property Editor.

Wwise decides which voices to keep according to which objects have the highest priority. So, those with the lowest priority that are in excess of the number of virtual voices will be virtualized or killed.
