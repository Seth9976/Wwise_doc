Plug-in version doesn't match sound engine version. Ensure the plug-in is compatible with this version of Wwise.

This error is a symptom of a bad compilation environment or, in Unity, bad integration. This always happen because of a recent Wwise upgrade. Plug-ins of any type (Effect, Source, etc) have a game-side library that needs to be linked with the game code. This error occur when the game-side plug-in code doesn't match the rest of the sound engine library. Note that the game binary could still link properly but not function at runtime.

Probable causes:

  • There is an old library that was not updated because of file permissions.

  • In Unity, there is an old DLL/SO/BUNDLE/PRX remaining in the Plugins folder, usually because of file permissions.

  • In Unity, there is an old DLL/SO/BUNDLE/PRX remaining on the console/mobile, preventing the replacement with the new file.

Recommended resolution steps:

  • Check the Wwise/SDK/[Platform]/lib folders and validate the libraries are all at the same timestamp.

  • In Unity, check the Wwise/Plugins/DSP folder and validate the libraries are all at the same timestamp.

  • Do a clean build of the game.

  • Clean the deployment folder on the target console/mobile, and redeploy the game.
