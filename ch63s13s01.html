Profiler Statistics (Query - Paths Used)

The 'Paths Used' query shows you which paths were resolved and added to the dynamic sequence list during the capture session, how many times they were resolved, and how many times this resulted in the playback of audio. Several filters also exist allowing you to sort through the information quickly to determine if any actions are necessary.

Interface Element

Description

Filters

Right-click the table header to open the Configure Columns dialog where you can specify which columns to display and their order. See Using tables.

Filter

The types of filters you can use to filter the information displayed in the Results pane. You can filter the information using any of the following filters:

  • Path - Displays information related to a particular path.

  • Added - Filters the information displayed in the Results pane based on the number of times a path was resolved and an audio structure was added to the dynamic sequence list.

  • Played - Filters the information displayed in the Results pane based on the number of times a path resulted in the playback of an audio structure.

Parameters

A series of properties that allow you to define the specific criteria for each filter.

For fields that accept text strings, you can type any portion of the name; however, wildcard symbols are not supported.

Results

Right-click the table header to open the Configure Columns dialog where you can specify which columns to display and their order. See Using tables.

Path

The State Groups and States that make up the path that was resolved during the capture session.

Added

The number of times the corresponding path was resolved and an audio structure was added to the dynamic sequence list.

Played

The number of times the resolved path resulted in the playback of an audio structure.
