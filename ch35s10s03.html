Assigning Conversion Settings ShareSets to Objects

After you have created the Conversion Settings ShareSets for your project, you can start assigning them to the objects in your project hierarchy. Conversion Settings ShareSets, like other object properties, are inherited from parent to child, which means that if you assign a Conversion Settings ShareSet to a Property Container, all containers and objects below it automatically use the same ShareSet. You can override the ShareSet used by the parent object. This allows you to apply a different Conversion Settings to specific objects and/or sources in the hierarchy.

By default, all Conversion Settings instances you create begin as ShareSets. However, you may want to tweak a ShareSet for one particular object. If you don't want to share the Conversion Settings between objects, you can create a custom instance of the ShareSet. When using a custom instance of a ShareSet, the settings are applied to the current object only. If any changes are made to the Conversion Settings only this object is affected.

To assign a Conversion Settings ShareSet to an Object:

  1. Load a top-level object into the Property Editor.

    Note

    If the object isn't a top-level object, you must enable the Override Parent option before you can set the Conversion Settings options.

  2. Switch to the Conversion category.

  3. In the Conversion Settings group, click the Set Conversion Settings button.

    The Project Explorer - Browser appears displaying a list of Conversion Settings ShareSets that are available in the project.

  4. From the list, select the ShareSet you want to apply to the object.

  5. If you want to apply a custom instance of the ShareSet to the object, click the three vertical dots and then select Define custom.

    The name of the Conversion Settings instance now has the word “Custom” appended to it. From now on, changes you make to the instance affects only the one object that uses it.

  6. Click the Open in a new window button to load the ShareSet or custom instance into the Conversion Settings Editor, where you can fine-tune the various Conversion Settings.

  7. In the Audio Sources table, review the list of audio sources to which these Conversion Settings will be applied to make sure that it's correct.

    The next step is to convert the audio sources. For more information on converting audio sources, refer to Converting audio files.
