Generated Files Tab

The Generated Files tab of the File Manager displays information about the files in the output folder(s) of your project. The Generated Files tab allows you to view the generated files for each of your project's platforms, one at a time.

The output folder(s) for your project are configured via the project's SoundBank Settings. Your project's output folders consist of one folder for project-level files and one folder for each of the project's platforms. By default, the project output folder is <Project>/GeneratedSoundBanks, and the platform folders are children of the project folder, named using the platform name. For example, given a project named MyProject that supports platforms Windows and Mac, the output folders look like:

  • <Project>/GeneratedSoundBanks

  • <Project>/GeneratedSoundBanks/Windows

  • <Project>/GeneratedSoundBanks/Mac

The files contained in the output folders is determined by the SoundBank Settings. For more information, refer to Soundbanks tab of the Project Settings dialog

In the File Manager, you can determine the status and owner of generated files, as well as which files are read-only. You can also move or delete the generated files you no longer need directly from within the File Manager. If, however, you are using a source control plug-in, an additional set of commands, such as Submit Changes and Check out, will be available by right-clicking a file from the list. The specific commands displayed depends on which source control system you are using with Wwise.

Interface element

Description

Platform

The platform for which the generated files are displayed.

Folder

The location or path of the folder in which the generated files are saved.

Opens a search field where standard alphanumeric entries filter out unmatching elements from the view. Learn more in Using tables.

Click the Close icon to the left of the search icon to close the search field and remove the filter.

Note

The searches do not include elements in collapsed nodes of the List View, Query Editor, MIDI Keymap Editor, and Reference View.

The search is not included in the Settings for Source objects.

Right-click the table header to open the Configure Columns dialog where you can specify which columns to display and their order. See Using tables.

File

The name of the files or folders.

Status

The status of the file, for example, edit or add. The status of the file comes from your source control software via the Wwise source control plug-in.

For example, if you are using the Perforce plug-in, your files can have any of the following statuses:

  • Add: The file is in the changelist and ready to be checked in to the depot.

  • Branch: A copy of an existing depot file has been added to the changelist and is ready to be checked in to the depot.

  • Delete: The file is in the changelist and is ready to be deleted from the depot.

  • Edit: A working copy of the depot file is checked out to your workspace for editing.

  • Local only: A file has been created locally and has not yet been checked in to the depot.

  • Lock: The depot file is locked so that no other user can check out the file for editing.

  • Normal: The file is checked in to the depot.

If the status of the file is unknown, a hyphen (-) is displayed.

Owner(s)

The owner(s) of the file.

If the owner of the file is unknown, a hyphen (-) is displayed.

Read-Only

Displays whether the file is read-only.

Copies the contents of the dialog to the Windows clipboard so that you can paste the information into a new file.

Closes the File Manager.
