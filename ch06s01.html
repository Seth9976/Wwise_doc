Customizing keyboard shortcuts and commands

To work faster, use keyboard shortcuts to type combinations of keys on your keyboard instead of clicking through the UI. Use the Keyboard Shortcut dialog to configure most of the keyboard shortcuts in Wwise. The available commands are listed, by category, in the Keyboard Shortcuts dialog with their associated shortcut key.

When you hover the mouse over a button or tool in the interface, a tooltip appears. The tooltip gives a description of the tool and often its associated keyboard shortcut. You can also see keyboard shortcuts in the menus.

Wwise also has other keyboard shortcuts not listed here because they are not editable.

To show the Keyboard Shortcuts and Commands dialog

  • In Wwise, click Project > Keyboard Shortcuts and Commands.

  • Press Ctrl+Shift+K (default).

Adding Keyboard Shortcuts

For each command in the Keyboard Shortcuts and Commands dialog, you can set one or many shortcut keys.

To assign a new shortcut to a command

  1. Select the command in the command list.

  2. Click the New Shortcut edit box to place the keyboard focus on it.

  3. Press your shortcut on the keyboard. For example, to assign Ctrl+B to a command, press and hold Ctrl and press B.

  4. If a conflict is shown, choose another shortcut key or remove the conflict.

  5. Click Assign.

To unassign an existing shortcut on a command

  1. Select the command in the command list on the left.

  2. Select the shortcut key you want to remove in the Current Shortcuts list.

  3. Click Remove.

Tip

To create a shortcut list for your own reference:

  1. Open the Keyboard Shortcuts and Commands dialog.

  2. Select the whole shortcut list (or press Ctrl+A).

  3. Press Ctrl+C to copy the shortcuts.

  4. Paste the tab-separated content in a spreadsheet application.

Adding Custom Commands from the Keyboard Shortcut Dialog

Create your own shortlist of custom commands that can be assigned to keyboard shortcuts to accommodate and speed up your workflows. This is done by saving your custom commands list to a .json file that is referenced by Wwise. Once loaded, your custom commands are displayed in the Keyboard Shortcut menu.

Wwise’s Keyboard Shortcuts menu facilitates creating, editing and reloading your custom commands.

To create a list of custom commands:

  1. Open the Keyboard Shortcuts and Commands dialog.

  2. Click Create. You are prompted to select the location to save your list. Options are:

    • Create in Installation Folder: In the installation folder, under Authoring\\Data\\Add-onsCommands.

    • Create in Project Folder: In the project folder, under Add-onsCommands.

    • Create in User Folder : In the user data directory, under %APPDATA%\\Audiokinetic\\Wwise\\Add-onsCommands.

  3. Select your preferred location. A file explorer window is opened and displays the default commands.json name for the file you’re about to create.

  4. Optionally rename the file and click Save. The .json file is saved at the specified location and is automatically opened in your default editor.

    Note

    There can only be one .json file per directory, used to define custom commands.

  5. Populate your list of custom commands and save the file. Refer to Defining Command Add-ons for information on how to define custom commands.

  6. In Wwise, click Reload All, to reload the latest changes to the .json file. Your custom commands are displayed in the list of commands where you can assign keyboard shortcuts to them.

To edit a list of custom commands:

  1. Open the Keyboard Shortcuts and Commands dialog.

  2. Click Edit. You are prompted to select the location of your .json file. Options are:

    • Edit in Installation Folder: In the installation folder, under Authoring\\Data\\Add-onsCommands.

    • Edit in Project Folder: In the project folder, under Add-onsCommands.

    • Edit in User Folder: In the user data directory, under %APPDATA%\\Audiokinetic\\Wwise\\Add-onsCommands.

  3. Select the appropriate location. A file explorer window is opened and displays the .json file you created earlier.

  4. Select the .json file and click Open. The file is opened in your default editor.

  5. Make your changes and save the file.

  6. In Wwise, click Reload All to reload the latest changes to the .json file. Your custom commands are displayed in the list of commands and you can assign keyboard shortcuts to them.

Interface element

Description

Opens a search field where standard alphanumeric entries filter out unmatching elements from the view. Learn more in Using tables.

Click the Close icon to the left of the search icon to close the search field and remove the filter.

Note

The searches do not include elements in collapsed nodes of the List View, Query Editor, MIDI Keymap Editor, and Reference View.

The search is not included in the Settings for Source objects.

Right-click the table header to open the Configure Columns dialog where you can specify which columns to display and their order. See Using tables.

Command List

Name

Displays the name or category of the command.

Shortcut

Displays the list of shortcuts currently assigned to a specific command.

Note

If the shortcut is grayed-out, it means the shortcut is read-only and is displayed for reference. It cannot be changed.

Command Add-ons

Create

Displays a menu allowing you to create a new command definition file at the specified location.

Edit

Displays a menu allowing you to edit an existing command definition file at the specified location.

Reload All

Reloads all Command Add-ons from the definition files found at all known locations. This is used to reload your custom commands after editing the definition files.

Current Shortcuts

Remove

Removes the currently selected shortcut(s) from the assignation list of the current command.

Reset to Default

Resets the current command to its factory default shortcut.

Caution

This action removes all shortcuts manually assigned to the current command.

New Shortcut

Edit-box to set a new shortcut for the current command. Once the key is entered, click Assign to bind the shortcut key to the current command.

Assign

Assign the shortcut displayed in the edit box to the current command.

Reset All Shortcuts

Resets all commands in the list to their factory default shortcuts.

Caution

This action removes all manually assigned shortcuts from all commands.

Conflict

The conflict box is only visible when a conflict occurs after assigning a shortcut. It shows which command is creating the conflict.

Remove [conflict]

When a conflict occurs after assigning a shortcut, the Remove button is displayed in the conflict box. Click Remove to remove (unassign) the shortcut from the conflicting command.

Go to conflict

Go to the conflicting command. After the conflict is resolved, you must return to the original command and re-type the shortcut again.
