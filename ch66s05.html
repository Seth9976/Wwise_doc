Motion

The Motion plug-in is a source plug-in that you can use to generate motion effects for the different devices that are used by your game, including game controllers. With the Motion plug-in, you can apply different Driver curves for each of the actuators for different motion devices used by your game. Using a possible combination of Driver offsets, State offsets, and RTPC curves, these eight Driver channels define the intensity and duration of the motion effects.

The Motion plug-in must be installed using the Launcher. For information on installing plug-ins, see Installing Plug-ins in the Audiokinetic Launcher guide.

For information on creating sources using plug-ins, see Creating sources using plug-ins.

Caution

Do not use high-pass filters on motion objects. Motion devices only produce low frequencies, so any high-pass filter effectively mutes the motion.

You may notice that certain property values in the Source Editor have an indicator beside them. This indicator shows if the property value is associated with a Game Parameter using RTPCs.

The following tables describes the two types of RTPC indicators:

Indicator

Name

Description

RTPC - On

A property value that is tied to an in-game parameter value using RTPCs.

RTPC - Off

A property value is not tied to an in-game parameter value.
