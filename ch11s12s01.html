Work Units Tab

The Work Units tab of the File Manager displays information about each of your Work Units, including information about the status and owner of the files, as well as which files are read-only.

You can also perform some project management tasks, such as file renaming and deletion. If, however, you are using a source control plug-in, an additional set of commands, such as check-in and check-out, are available by right-clicking the Wwise project file or a Work Unit in the list. The specific commands that will be displayed depend on which source control system you are using with Wwise.

Caution

The Default Work Units are critical project files and should not be renamed or deleted.

Interface element

Description

Perforce:

Subversion (deprecated):

Sends pending files to the server for processing. You can select any of the following options:

  • Work Units: Sends only the pending Work Units to the server for processing.

  • Sources: Sends only the pending source files to the server for processing.

  • All: Sends both the pending Work Units and source files to the server for processing.

This button is only available when you are using a source control plug-in.

Perforce:

Subversion (deprecated):

Updates the working copy of your files by retrieving the latest version of the files from the depot. You can select any of the following options:

  • Work Units: Updates only the Work Unit files by retrieving the latest version of the files from the depot.

  • Sources: Updates only the source files by retrieving the latest version of the files from the depot.

  • All: Updates both the Work Units and source files by retrieving the latest version of the files from the depot.

This button is only available when you are using a source control plug-in.

Project Folder

The location or path of the folder in which the current project is saved.

Opens a search field where standard alphanumeric entries filter out unmatching elements from the view. Learn more in Using tables.

Click the Close icon to the left of the search icon to close the search field and remove the filter.

Note

The searches do not include elements in collapsed nodes of the List View, Query Editor, MIDI Keymap Editor, and Reference View.

The search is not included in the Settings for Source objects.

Right-click the table header to open the Configure Columns dialog where you can specify which columns to display and their order. See Using tables.

File

The name of the current file or folder. The file can be a Wwise project file, a Work Unit file, or a folder.

Status

The source control status of the file, for example, edit or add. The status of the file comes from your source control software via the Wwise source control plug-in.

For example, if you are using the Perforce plug-in, your files can have any one of the following statuses:

  • Add: The file is in the changelist and ready to be checked in to the depot.

  • Branch: A copy of an existing depot file has been added to the changelist and is ready to be checked in to the depot.

  • Delete: The file is in the changelist and is ready to be deleted from the depot.

  • Edit: A working copy of the depot file is checked out to your workspace for editing.

  • Local only: A file has been created locally and has not yet been checked in to the depot.

  • Lock: The depot file is locked so that no other user can check out the file for editing.

  • Normal: The file is checked in to the depot.

If the status of the file is unknown, a hyphen (-) is displayed.

Owner(s)

The owner of the file.

If the owner of the file is unknown, a hyphen (-) is displayed.

Modified

Displays whether the file has been modified.

Read-Only

Displays whether the file is read-only.

Copies the contents of the dialog to the Windows clipboard so that you can paste the information into a new file.

Closes the File Manager.
