Haptic Clip Player

The Haptic Clip Player is a source plug-in that enables you to play back parametric haptic content stored in .haptic files across haptic-enabled devices. It converts high-level haptic descriptions — amplitude envelopes, frequency envelopes, and transient events — into device-specific PCM waveforms using the Meta Haptics SDK.

The Haptic Clip Player plug-in is part of the Motion plug-ins. It must be installed using the Launcher. For information on installing plug-ins, see Installing Plug-ins in the Audiokinetic Launcher guide.

For information on creating sources using plug-ins, see Creating sources using plug-ins.

At runtime, the Haptic Clip Player automatically detects the connected haptic device and selects one of three rendering profiles to optimize playback quality. If the device is not yet available when the source is initialized, the plug-in defaults to the Generic Low-Resolution Haptics profile and re-evaluates on the next audio cycle.

  • Generic Rumble: Traditional rumble motors with simple amplitude control. Supported on Xbox (XInput), PlayStation 4, and GameInput Rumble devices.

  • Generic Low-Resolution Haptics: Mid-range linear actuators capable of varied waveforms. Supported on iOS (CoreHaptics) and Nintendo Switch (HD Rumble).

  • Generic High-Resolution Haptics: Advanced actuators with wide frequency response and precise waveform control. Supported on PlayStation 5 DualSense and GameInput Haptics devices.

  • Meta Quest Haptics: Dedicated profile for Meta Quest controllers, tuned specifically for the haptic actuators used in Meta Quest hardware.

Note

The Haptic Clip Player does not currently support RTPCs or looping. To loop haptic content, use a Wwise container.
