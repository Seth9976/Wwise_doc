AK Custom Panner

You can use the AK Custom Panner plug-in to create custom speaker position configurations for audio busses. The plug-in uses a Vector Base Amplitude Panning (VBAP) algorithm, which supports customized panning and sound delivery to non-standard audio configurations.

The Custom Panner acts as an out-of-place object processor that can output a different number of channels than its input channel configuration. See Implementing an Out-of-Place Object Processor for more information.

Installation

To use the Custom Panner, you must install it through the Audiokinetic Launcher. See Installing Plug-ins for more information. The plug-in requires a license to be activated and for SoundBanks to be generated successfully. You can acquire a free license from the Audiokinetic website at Plug-ins: AK Custom Panner.

After the plug-in is installed, the Custom Panner factory assets are added to new projects by default. To add them to existing projects, use the Import Factory Assets dialog.

Using the Customer Panner

Before you insert the Custom Panner on a bus, set the Bus Configuration to Audio Objects. See Property Editor: Audio Bus for details.

By default, the Custom Panner sound output is sent as a single anonymous multi-channel audio object. If a speaker is marked as LFE, the Custom Panner mixes the input LFE content into the single audio object, and that speaker is not included in the positioning algorithm (VBAP). Alternatively, when Object Mode is selected, the sound output is sent as multiple mono audio objects. If a speaker is marked as LFE, the Custom Panner sends an additional audio object with an LFE configuration (channel configuration of 0.1).

Muting and soloing speakers

You can use the Mute (M) and Solo (S) buttons to silence or isolate speakers, respectively. However, the behavior varies slightly depending on whether the affected speakers are real or virtual:

  • Muting speakers prevents them from emitting any sound, including contributions from virtual speakers.

    Muting virtual speakers stops their contributions to any real speakers.

  • Soloing a speaker mutes all other real speakers. The solo speaker emits its direct sound as well as any contributions from virtual speakers.

    Soloing a virtual speaker mutes all direct sound to real speakers and all contributions from other virtual speakers. Soloing a virtual speaker is a way to isolate the sound it redistributes to real speakers.

Using virtual speakers

You can use the Custom Panner to set selected speaker positions as virtual. A virtual speaker is a position included in the VBAP triangulation but does not correspond to a physical output.

When audio is panned to a virtual speaker, the plug-in redistributes that signal's energy to the closest real speakers in the triangulation, which sound designers can use to manage complex spatialization. Through strategic virtual speaker placement, you can increase panning behavior predictability, especially in highly irregular speaker configurations in which VBAP-only calculations might otherwise cause discontinuities in panning.

Caution

For the redistribution algorithm to work properly, virtual speakers must be surrounded by real speakers. Configurations of multiple virtual speakers as direct neighbors of one another are not supported.

AK Custom Panner properties

Interface Element

Description

Adds a speaker position to the list.

Removes the selected speaker positions from the list.

Object Mode

Determines the number of audio objects the plug-in creates. If selected, the plug-in creates as many objects as speaker positions. If not selected, the plug-in creates a single object with as many channels as the number of speaker positions.

Channel

Output channel number assigned to the speaker position.

This value is displayed as "-" if the speaker is marked as Virtual.

[M]

(Mute) Mutes the corresponding speaker position, which can be helpful during speaker configuration.

You can mute multiple speaker positions simultaneously.

[S]

(Solo) Mutes all other speaker positions so that you can audition the selected speaker position in isolation.

You can set more than one speaker position to solo. To isolate a single speaker position when other speaker positions are set to solo, Ctrl+click the Solo button to select the current Solo button and clear all other Solo buttons.

Name

Name of the speaker position.

LFE

Removes the speaker position from the panning algorithm and assigns it to the LFE channel.

Virtual

Marks the speaker position as virtual.

When selected, the speaker is not included in the output of this plug-in. The speaker output is redistributed to its closest non-virtual neighbors on the VBAP sphere.

Caution

Avoid configuring multiple virtual speakers as direct neighbors of one other. This type of configuration is not supported and can cause unpredictable panning behavior.

Azimuth

Speaker position horizontal plane azimuth.

When set to 0, the speaker position is directly in front. Negative angles move the speaker position to the left. Positive angles move the speaker's position to the right.

Default value: 0

Range: -180 to 180

Units: Degrees

Elevation

Speaker position vertical plane elevation.

When set to 0, the speaker position is directly in front. Positive angles move the speaker position upward. Negative angles move the speaker position downward.

Default value: 0

Range: -90 to 90

Units: Degrees
