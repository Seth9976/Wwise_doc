Connecting to a local/remote game system

If you want to simulate different sounds in game, or want to profile and troubleshoot different aspects of your game on a particular platform, you need to first connect to the machine upon which the game runs. You can connect to any Wwise sound engine that is running and available on your local area network.

Caution

When profiling, it is recommended that you connect to the Profile build configuration of the Wwise sound engine, even from the Debug build of your game, because the performance of the Debug configuration has not been optimized.

To help you find the machine you are looking for, Wwise automatically searches for all PCs and game consoles on the same subnet of the network that are currently running a version of the Wwise sound engine. You can also connect to consoles or PCs outside your subnet by manually entering the IP address of the platform.

Note

If your game doesn't appear in the Remote Connections dialog in Wwise, refer to Troubleshooting remote connection issues to troubleshoot the problem.

To connect to a local/remote game system:

  1. On the toolbar, click the Connect to Remote Platform button. See Remote Connections dialog.

    The Remote Connection dialog opens with a list of game systems on the network that are currently running a version of the Wwise sound engine.

  2. From the list, select the platform to which you want to connect.

    Note

    The status of the machine must be “Ready” before you can connect to it.

  3. If you want to start capturing information from the moment you connect, select the Start capture on connect option.

  4. Click Connect.

    Wwise connects to the local or remote platform. The name of the machine appears in the toolbar.

    Note

    You can manually connect to a platform by clicking Connect To IP and then typing the IP address of the remote machine.

To disconnect from a local/remote machine:

  1. On the toolbar, click the Disconnect from Remote Platform button.

    Wwise disconnects from the machine.

Using the Remote Connection history list

Wwise maintains a history of all the machines/systems that you've successfully connected to in the past. This way you can easily find a system that you've been working on. Before connecting to a machine, you must first verify its status. The status of a machine can be one of the following types:

  • Ready: When it is ready to accept a connection.

  • Busy: When it is already connected to a machine and is therefore not accepting any connections.

  • Different Version: When the version of Wwise is not compatible with the version of the game running on the remote system.

  • Not Available: When it is no longer connected to the network.

You can clean up the history when you no longer want machines to appear in your history list.

To connect to a system using the history list:

  1. On the toolbar, click the Connect to Remote Platform button.

    The Remote Connection dialog opens with a list of machines that are currently running on the network.

  2. Switch to the History tab.

  3. From the History list, select the machine to which you want to connect.

  4. If you want to start capturing information from the moment you connect, select the Start capture on connect option.

  5. Click Connect.

    Wwise connects to the local/remote machine. The name of the machine appears in the toolbar.

To clean up your history list:

  1. On the toolbar, click the Connect to Remote Platform button.

    The Remote Connection dialog opens with a list of machines that are currently running on the network.

  2. Switch to the History tab.

  3. From the History list, select the machine that you want to remove.

  4. Do one of the following:

    • Click Remove.

    • Press the Delete key.

    The machine is removed from the History list.
