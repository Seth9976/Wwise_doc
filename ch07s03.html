Defining your project settings

In Wwise, you can define certain settings for your project in the Project Settings dialog. In the Project Settings dialog, you can specify certain settings and default values for your project. These settings and default values and properties that you define will be used by the project until you edit or override these settings. The following pages detail the different tabs in the dialog and their associated project settings.

The Project Settings dialog includes the following tabs:

  • General to define a source control plug-in, the location for the Originals folder for your project assets, automatic sample rate detection settings, and Event name creation settings.

  • Conversion to set the default Conversion Settings ShareSet for the project as well as the automatic sample rate detection settings.

  • SoundBanks to define the SoundBank settings for your project, which includes whether to generate content files, a header file, and the maximum attenuation information for Events with your SoundBanks, whether SoundBank names can be used, the location where your SoundBank will be saved, and optionally, user-defined custom steps that will be performed before and/or after SoundBank generation.

  • Logs to manage the warnings, errors, and messages that Wwise displays in the Conversion and SoundBank logs.

  • Environmental Curves to define the default volume, LPF, HPF, and DSF curves for obstruction, occlusion, diffraction, and transmission in your project.

  • External Sources to specify the input and output paths for the external audio sources that will be used with the External Source plug-in.

  • Network to specify port numbers to be used during communication between the authoring application and your game.

  • Defining Custom Properties to define properties for the Sound and Audio Source objects of the Containers hierarchy.

Note

Wwise applies any changes made to your Project Settings only after you click OK. Live editing isn't possible.

You can configure volume threshold and max voice instances for each platform. See Managing platforms.
